## 2. 이동 & 카메라

### 2-1. 플레이어 이동
- **방향:** 4방향 (상하좌우)
- **입력:** WASD (추후 게임패드 지원)
- **공격 방향:** 플레이어 캐릭터 화면 중앙 기준 마우스 커서 좌표 방향
- **속도:** 조절 가능한 export 변수 (프로토타입 기본값 테스트 후 확정)

### 2-3. 입력 키 매핑
| 조작 | 입력 |
|------|------|
| 이동 | WASD |
| 확인/대화 | F |
| 취소/뒤로 | ESC |
| 공격 | 마우스 좌클릭 (커서 방향으로) |
| 구르기 (회피) | SpaceBar |
| 패링 (막기) | 마우스 우클릭 |
| 대화 넘기기 | SpaceBar / 마우스 우클릭 / Enter |

### 2-2. 카메라
- **기본:** 플레이어 고정 추적 (Camera2D, smoothing 적용)
- **전투 시:** 카메라 고정 (전투 영역 중심으로 lock)
- **맵 경계:** 카메라 리밋으로 맵 밖 노출 방지

## 3. NPC 대화 시스템

### 3-1. 대화 UI
- **스타일:** 하단 텍스트박스 (화면 하단 고정)
- **표시 정보:** 화자 이름 + 대사 텍스트
- **텍스트 연출:** 타이핑 효과 (글자 순차 표시), 스킵 가능

### 3-2. 선택지
- **표시 방식:** 대화 중 인라인 (텍스트박스 내부/상단에 선택지 나열)
- **최대 개수:** 시스템 상 5개 지원, 연출 권장 3개 이하
- **입력:** 방향키로 커서 이동 + 확인 키

### 3-3. 대화 데이터
- **포맷:** JSON
- **구조 (초안):**
```json
{
  "dialogue_id": "npc_001_greeting",
  "lines": [
    {
      "speaker": "Miren",
      "text": "외눈박이가 여기엔 웬일이야?",
      "next": "npc_001_choice"
    }
  ],
  "choices": {
    "npc_001_choice": {
      "prompt": "",
      "options": [
        { "text": "그냥 지나가는 길이야.", "next": "npc_001_neutral", "flags": {} },
        { "text": "네가 상관할 일이 아니지.", "next": "npc_001_hostile", "flags": { "reputation_twoeye": -1 } },
        { "text": "이 마을이 궁금했어.", "next": "npc_001_friendly", "flags": { "reputation_twoeye": 1 } }
      ]
    }
  }
}
```
- 선택지의 `flags`로 평판/이벤트 플래그 연동
- 추후 연출이 필요해지면 Dialogic 등 에디터 도입 고려

## 4. 전투 시스템

### 4-1. 인카운터
- **방식:** 심볼 인카운터 — 필드의 적 오브젝트와 접촉 시 전투 시작
- **전투 공간:** 필드 위에서 바로 진행 (씬 전환 없음)
- **전투 시작 시:** 카메라 고정 + 구경꾼 NPC 배치

### 4-2. 플레이어 조작
| 조작 | 설명 |
|------|------|
| **기본 공격** | 무기 휘두르기. 스킬 없음. |
| **구르기 (회피)** | 스태미나 소모. 무적 프레임. 반격 없음 — 순수 회피용. |
| **막기 (패링)** | 스태미나 소모. 저스트 타이밍 시 반격기 즉발. |

- **스태미나:** 시간 경과로 자연 회복. UI 표시 방식은 프로토타입에서 테스트.

### 4-3. 적 공격 예고
- **눈 모양 게이지바:** 적 머리 위에 눈 형태 게이지 표시
- 게이지가 차오르며 눈이 서서히 열림 → 완전히 뜨는 순간 공격 발동
- 게이지 속도로 난이도 조절 (느릴수록 쉬움)

### 4-4. 체력 시스템
- **플레이어:** 하트 시스템. Hit 1회 = 하트 1개 차감. 숫자 스탯 없음.
- **적 (잡몹):** 하트 시스템 동일. HP바 미표시 — 피격 시 리소스 바리에이션으로 연출 (스프라이트 변화, 흔들림 등).
- **적 (보스):** 별도 고려 (Phase 2 이후)

### 4-5. NPC 참전 시스템
- **전투 영역:** 카메라 화면에 비치는 맵 전체
- **구경꾼:** 전투 범위 내 NPC가 주위를 둘러쌈
- **참전 조건:** 구경꾼의 소속 세력 평판 기반
- **참전 수 제한:** 최대 5명 (테이블에서 조절 가능)

### 4-6. 난이도 포지션
- 스토리 진행용 허들. 재밌되 어렵지 않게.
- 구르기만으로 충분히 클리어 가능, 패링 반격은 숙련자 보상.

## 5. 기반 시스템

### 5-1. 평판 시스템
- 지역별 int 값 (예: `reputation_oneeye`, `reputation_twoeye`, `reputation_trade`)
- 대화 선택, 전투 결과, 퀘스트 등으로 증감
- 평판 단계: 적대 / 경계 / 중립 / 호감 / 신뢰 (컨셉 문서 3-5 참조)

### 5-2. 이벤트 플래그 시스템
- Dictionary 기반 (`event_flags: { "met_lume": true, "beat_bully_ch1": true }`)
- NPC 대사 분기, 참전 조건 등에 활용
- 구현 우선순위 1순위 (컨셉 문서 기준)

### 5-3. 세이브/로드
- Godot ResourceSaver/Loader 또는 JSON 파일 기반
- 저장 데이터: 플레이어 위치, 평판 값, 이벤트 플래그, 하트 수, 현재 챕터
- 프로토타입에서는 수동 세이브 1슬롯으로 시작

## 6. 씬 전환

### 6-1. 맵 간 이동
- **방식:** 출입구 트리거 (Area2D)
- 플레이어가 출입구 영역에 진입하면 전환 실행
- 문 오브젝트: 상호작용 키 → 문 열기 애니메이션 → 전환

### 6-2. 전환 효과
- **기본:** 페이드 아웃 → 씬 로드 → 페이드 인
- 문 열기 시: 애니메이션 재생 → 페이드 전환 (쫀득한 연출)

## 7. Phase 1 완료 조건

플레이 가능한 빌드에서 다음이 동작:
1. ✅ 4방향 탑다운 이동 + 카메라 추적
2. ✅ NPC와 대화 (하단 텍스트박스 + 선택지)
3. ✅ 심볼 인카운터 → 필드 전투 (공격/회피/패링)
4. ✅ 선택에 따라 NPC 반응 변화 (평판 + 이벤트 플래그)
5. ✅ 씬 전환 (출입구 트리거 + 페이드)
6. ✅ 세이브/로드

**그레이박스 콘텐츠:** Ch0 외눈박이 마을 1맵, NPC 3~5명 (플레이스홀더 대사), 전투 적 1~2개
